# Game Vision

*Source: `raw/` → SESSION_HANDOFF.md (prior session notes).*

A casual 2D mobile game where players slash falling Arabic letters, Fruit Ninja–style: swipe trail, hits, combos, bombs, game over.

## Builder profile
Solo dev; strong full-stack/enterprise + Flutter; limited game dev experience. Hobby/portfolio pace, not time-critical.

## Constraints & preferences
- Platform: Flutter, Android + iOS (web was considered earlier; mobile-first chosen).
- Backend: none preferred; Firebase only if truly needed.
- Scope: MVP first (slash mechanic, scoring, game over), then modes/progression.

## Open questions
- Is Arabic purely aesthetic, or does it drive mechanics (target letters, word-forming, educational value)?
- Persistence approach for high scores/progress without a backend.
- Monetization/shareability (possible tie-in to Talab drinks brand — optional).
- Realistic timeline and biggest technical risks (game loop, physics, Arabic text rendering).
